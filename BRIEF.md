# Aurion Mandala — Website Brief

## Stage
Pre-revenue, proof-of-concept transitioning into design-partner phase.

## Audience
- Investors
- Enterprise partners
- Strategic collaborators

## Core Positioning
**Selective access to an emerging infrastructure layer.**

This is technically real, thoroughly designed, and ahead of the curve.

## What NOT to do
- Do NOT present it like a SaaS product
- Do NOT push onboarding or usage flows
- Do NOT pretend scale exists
- Do NOT over-explain the full system
- No pricing, no feature lists, no "sign up free"
- No markdown-style busy layouts

## Primary CTAs (3 only)
1. Request Access
2. Apply as Design Partner
3. Institutional Inquiry (or "Partner Inquiry")

## Must-Have Proof Element
Show at least ONE visual proof that this is beyond conceptual:
- Architecture diagram (placeholder for now), OR
- SVA → Mandala transformation example, OR
- Verification flow diagram

Use a placeholder/mock-up diagram with clean line art style. Label it clearly.

## Key Copy Line
> "Currently working with select partners to deploy and stress-test SVA infrastructure."

## Design Direction
- Dark, premium, infrastructural — think Stripe, Linear, Vercel but more selective and mysterious
- Deep blacks / near-blacks with subtle warm or cool tint
- One strong accent color (deep gold OR electric blue — choose what works best)
- Clean serif for headings (e.g. Playfair Display or similar), clean sans for body
- No gradients unless very subtle
- Generous whitespace — the page should breathe
- Minimal animations: subtle fade-ins only
- Mobile responsive

## Page Structure (single long-scroll page)

### 1. Hero
- Logo placeholder (top left)
- Nav: About / Proof of Concept / [Apply as Design Partner button]
- Headline: Something like "Infrastructure for Verified Identity at Scale"
- Sub-headline: 1-2 sentences on what Mandala is
- 2 CTAs: "Request Access" and "Apply as Design Partner"
- Subtle background texture or grid

### 2. What is Aurion Mandala
- 2-3 short paragraphs explaining the system (SVA = Structured Verifiable Attributes, Mandala = the transformation/verification layer)
- Keep it conceptual-but-credible, not a product pitch
- Include one key differentiator line

### 3. Proof of Concept (the technical credibility section)
- Section title: "Beyond Conceptual"
- The key copy line: "Currently working with select partners to deploy and stress-test SVA infrastructure."
- A clean diagram placeholder — use a simple SVG/HTML diagram showing:
  - Input: SVA Data Layer
  - Process: Mandala Transformation Engine
  - Output: Verified Identity Record
  - Make it look like a real architecture diagram with labeled boxes and arrows
- 2-3 bullet points on what has been validated/built

### 4. Who We Work With (Partner Criteria)
- Not "anyone can apply" — selective framing
- 3 columns: Investors / Enterprise Partners / Strategic Collaborators
- Each with a 2-3 line description of what kind of partner fits

### 5. CTA Section

- 3 buttons: Request Access / Apply as Design Partner / Institutional Inquiry
- Each button links to a simple mailto or anchor (use # for now)
- Brief copy under each CTA explaining what it's for
